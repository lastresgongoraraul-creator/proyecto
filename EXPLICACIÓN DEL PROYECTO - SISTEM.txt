EXPLICACIÓN DEL PROYECTO - SISTEMA DE RECOMENDACIÓN DE VIDEOJUEGOS.

1. Arquitectura General
El proyecto utiliza una arquitectura de microservicios dockerizados, lo que permite que cada parte del sistema sea independiente y escalable.

2. Descripción de Carpetas y Servicios

/backend (Servicio Principal - Java/Spring Boot)

Es el "corazón" del sistema. Se encarga de toda la lógica de negocio, seguridad y gestión de datos.
-Función: Gestiona los usuarios (registro/login), la persistencia de reseñas y el catálogo de juegos.
- Tecnologías: Spring Boot 3, Spring Security (JWT), Hibernate/JPA.
- Destacado: Implementa un sistema de bloqueos en la base de datos para asegurar que el cálculo de las notas medias de los juegos sea exacto y no haya errores cuando muchos usuarios escriben reseñas a la vez.

/frontend (Interfaz de Usuario - React/TypeScript)
Es lo que el usuario ve y con lo que interactúa.
- Función: Permite navegar por el catálogo, buscar juegos por género o plataforma, leer y escribir reseñas, y ver recomendaciones de la IA.
- Tecnologías:React, Vite, Tailwind CSS, React Query (para gestión de estado y peticiones).
- Diseño: Estética moderna "Dark Mode" con micro-animaciones y diseño responsivo para móviles y PC.

/ai-service (Motor de Inteligencia Artificial - Python/FastAPI)
Es el cerebro encargado de las recomendaciones inteligentes.
-Función: Convierte las descripciones de los juegos en vectores numéricos (embeddings). Cuando estás viendo un juego, este servicio busca en la base de datos otros juegos que tengan "vectores" similares para recomendarte títulos parecidos.
- Tecnologías: Python, FastAPI, PyTorch, Transformers (modelo all-MiniLM-L6-v2).

/chat-service (Comunicación en Tiempo Real - Node.js)
Servicio diseñado para futuras funcionalidades de interacción entre usuarios.
- Función: Gestionar conexiones WebSocket para actualizaciones en tiempo real.
- Tecnologías: Node.js, Socket.io.

/postgres (Base de Datos)
Base de datos relacional de alto rendimiento.
- Función: Almacena usuarios, juegos y reseñas. 
- Destacado: Utiliza la extensión pgvector para permitir búsquedas vectoriales por IA directamente en la base de datos.

/redis (Caché)
- Función: Mejora la velocidad del sistema almacenando sesiones y datos temporales para que el backend no tenga que consultar la base de datos principal constantemente.

---

3. Funcionamiento al Iniciar el Programa
Cuando ejecutas el proyecto (docker compose up), sucede lo siguiente:
1. Se levantan las bases de datos (Postgres y Redis).
2. El Backend se conecta a la base de datos y ejecuta las migraciones (scripts SQL que crean las tablas y los datos de prueba automáticamente).
3. El AI-Service carga el modelo de lenguaje en memoria para estar listo para procesar recomendaciones.
4. El Frontend se vuelve accesible en el navegador (generalmente en `http://localhost`).

---

4. Flujo de Usuario Típico
1. El usuario entra al catálogo y ve los juegos disponibles (datos servidos por el Backend).
2. El usuario hace click en un juego y, además de los detalles básicos, el sistema le muestra Juegos Similares (calculados por el AI-Service).
3. El usuario inicia sesión (autenticación gestionada por Spring Security con tokens JWT).
4. El usuario escribe una reseña; el Backend la guarda y actualiza instantáneamente la nota media del juego en la base de datos.
